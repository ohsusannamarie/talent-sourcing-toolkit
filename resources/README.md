## Recommended workflow

1. Pick a use case from [Starter Packs](./starter-packs.md).
2. Search the original source files using the keywords listed in that section.
3. Vet anything browser-based using [Chrome Extension Vetting](./chrome-extension-vetting.md).
4. Tag and organize useful finds with [Categories & Tags](./categories.md).
5. If you want a cleaner GitHub-native version, run the conversion script in [`scripts/`](../scripts/convert_resources_to_markdown.py).

---

## Why this exists

Huge resource lists are only useful if people can find their way through them. This folder is the map, the caution tape, and the friendly little sign that says, "Start here before you disappear into 1,885 rows of sourcing chaos."
