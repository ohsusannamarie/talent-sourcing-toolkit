# Audit a Sourcing Stack

Use this workflow when reviewing a personal, team, or company sourcing tool stack.

---

## Goal

Identify which tools are useful, redundant, risky, outdated, underused, or worth standardizing.

---

## Inputs

- Current sourcing tools
- Browser extensions
- Paid platforms
- Free tools
- Spreadsheets and trackers
- Outreach tools
- Team workflows
- Security or policy constraints

---

## Workflow

1. List every tool currently used.
2. Categorize by use case.
3. Mark free, freemium, paid, or unknown.
4. Identify duplicate functionality.
5. Check update history and vendor credibility.
6. Review privacy and permission concerns.
7. Identify tools worth keeping, replacing, or removing.
8. Build a recommended team stack.

---

## Suggested categories

- Candidate discovery
- Contact finding
- Profile enrichment
- Search and Boolean
- Outreach
- CRM / pipeline management
- Talent intelligence
- Productivity
- Browser workflow
- Data cleanup
- AI support
- Risk / caution zone

---

## Output

A prioritized sourcing stack recommendation:

- Keep
- Test
- Replace
- Remove
- Needs security review
- Needs pricing review

---

## Practical rule

A stack is only good if people understand when to use each tool and when not to.
