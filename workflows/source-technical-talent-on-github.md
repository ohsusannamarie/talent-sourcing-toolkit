# Source Technical Talent on GitHub

Use this workflow when searching for engineers, open-source contributors, maintainers, technical writers, or people working with specific tools and technologies.

---

## Goal

Find public technical signals that help identify relevant talent and support thoughtful, specific outreach.

---

## Inputs

- Target role
- Required technologies
- Adjacent technologies
- Preferred industries or domains
- Location constraints
- Seniority level
- Example companies or projects

---

## Workflow

1. Translate the role into technical keywords.
2. Search repositories by technology, topic, and project type.
3. Review contributors, maintainers, issues, and pull requests.
4. Look for pinned repositories and recent activity.
5. Validate relevance with multiple public signals.
6. Cross-reference professional links when available.
7. Personalize outreach around relevant work, not random personal details.

---

## Suggested search angles

- Language + domain
- Framework + project type
- Tool name + configuration file
- Company open-source project + contributors
- Topic tag + location clues
- Technical conference repo + contributors

---

## Output

A shortlist of potential technical candidates or communities with notes on why each signal matters.
