**Formerly Talent Sourcing Toolkit.**

**1,395 Google Custom Search Engines. 490 Chrome extensions. Real sourcing workflows. Less tab chaos.**

A practical command center for modern sourcers, recruiters, talent intelligence professionals, and recruiting teams who want better tools, sharper workflows, and fewer “why do I have 47 tabs open?” moments.

## What this is

This repo started as a sourcing resource archive: **1,395 Google Custom Search Engines** and **490 Chrome extensions** collected for talent sourcing and recruiting research.

It is now evolving into a broader command center for modern recruiting work.

Use it to:

- Find sourcing tools and search engines
- Build better Boolean, X-ray, and OSINT-style workflows
- Explore Chrome extensions with a safety-first lens
- Research technical talent, companies, communities, and markets
- Build talent intelligence and recruiter enablement resources
- Use AI thoughtfully in sourcing, outreach, research, and workflow design
- Create your own recruiter operating system

The goal is not to collect every recruiting link on the internet. The goal is to make useful recruiting resources easier to find, evaluate, and apply.

## How to use this repo

### 1. Start with a use case

Do not try to consume the whole thing at once. That way lies madness.

Start with a specific sourcing question:

- I need better X-ray search engines.
- I want Chrome extensions for recruiter productivity.
- I need tools for technical sourcing.
- I want to audit my current sourcing stack.
- I need to map a talent market.
- I want AI to help with research, outreach, or intake.
- I am building a sourcing training resource for my team.

### 2. Pick the right section

| If you need to... | Start here |
|---|---|
| Find people across the web | Google CSE archive + sourcing workflows |
| Source engineers or technical communities | [GitHub Sourcing](./docs/github-sourcing.md) |
| Evaluate browser tools | Chrome extension archive + [Productivity](./docs/recruiter-productivity.md) |
| Build market context | [Talent Intelligence](./docs/talent-intelligence.md) |
| Use AI without being reckless | [AI Recruiting](./docs/ai-recruiting.md) + [Compliance & Ethics](./docs/compliance-and-ethics.md) |
| Create team enablement | [Templates](./templates/) + [Workflow Recipes](./workflows/) |

### 3. Search the files

Use Excel, Word, or GitHub search to look for keywords like:

- `LinkedIn`
- `GitHub`
- `email`
- `profile`
- `resume`
- `boolean`
- `developer`
- `research`
- `automation`
- `export`
- `AI`
- `market map`
- `extension`

### 4. Test before adopting

Some tools may be outdated, deprecated, acquired, abandoned, blocked by your company's security policy, or risky from a privacy/compliance standpoint.

Treat this as a discovery library, not a pre-approved vendor list.

### 5. Build your own shortlist

The real value is not using everything.

The real value is finding the few tools, searches, and workflows that actually make your sourcing work faster, cleaner, smarter, or more human.

## Chrome extension safety checklist

Before installing any Chrome extension, especially for recruiting or research work, check:

- When it was last updated
- Who built it
- Whether the developer looks legitimate
- Number and quality of reviews
- Requested permissions
- Whether it can read or change data on websites
- Whether it touches LinkedIn, email, cookies, contacts, or candidate data
- Whether your company allows it
- Whether there is a privacy policy

A useful sourcing tool is not useful if it turns into a security problem. Be curious, but do not be reckless.

## AI recruiting caution

AI can help recruiters research faster, draft cleaner, summarize intake notes, expand search strings, compare markets, and personalize outreach.

AI should not be treated as a neutral decision-maker.

Be especially careful with tools that:

- Rank candidates
- Score resumes
- Recommend who should move forward
- Reject candidates automatically
- Infer protected or sensitive traits
- Scrape or enrich candidate data without clear consent or policy coverage

Use AI as decision support, not decision replacement.
