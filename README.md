## What this is

This repo is a sourcing resource archive and field guide. It is designed to help recruiting and talent teams explore tools, build research workflows, discover sourcing shortcuts, and create their own recruiter enablement libraries.

It is not meant to be a polished SaaS product. It is more like a giant, useful, slightly chaotic sourcing toolbox that has been cleaned up enough to be useful to other people.

## How to use this repo

### 1. Start with a use case

Do not try to consume the whole thing at once. That way lies madness.

Start with a specific sourcing question:

- I need better X-ray search engines.
- I want Chrome extensions for recruiter productivity.
- I need tools for technical sourcing.
- I want to audit my current sourcing stack.
- I am building a sourcing training resource for my team.

### 2. Search the files

Use the file search/filter tools in Excel or Word to find keywords like:

- `LinkedIn`
- `GitHub`
- `email`
- `profile`
- `resume`
- `boolean`
- `developer`
- `research`
- `automation`
- `export`

### 3. Test before adopting

Some tools may be outdated, deprecated, acquired, abandoned, or blocked by your company's security policy. Treat this as a discovery library, not a pre-approved vendor list.

### 4. Build your own shortlist

The real value is not using everything. The real value is finding the few tools that actually make your workflow faster, cleaner, or smarter.

## Chrome extension safety checklist

Before installing any Chrome extension, especially for recruiting or research work, check:

- When it was last updated
- Who built it
- Whether the developer looks legitimate
- Number and quality of reviews
- Requested permissions
- Whether it can read or change data on websites
- Whether it touches LinkedIn, email, cookies, contacts, or candidate data
- Whether your company allows it
- Whether there is a privacy policy

A useful sourcing tool is not useful if it turns into a security problem. Be curious, but do not be reckless.
