## What this is

This repo started as a sourcing resource archive: **1,395 Google Custom Search Engines** and **490 Chrome extensions** collected for talent sourcing and recruiting research. It is now a broader command center for modern recruiting work.

Use it to:

- Find sourcing tools and search engines
- Build better Boolean, X-ray, and OSINT-style workflows
- Explore Chrome extensions with a safety-first lens
- Research technical talent, companies, communities, and markets
- Build talent intelligence and recruiter enablement resources
- Use AI thoughtfully in sourcing, outreach, research, and workflow design
- Create your own recruiter operating system

The goal is not to collect every recruiting link on the internet. The goal is to make useful recruiting resources easier to find, evaluate, and apply.

## The searchable library

The Resource Library is now **data-driven**. One source of truth (`data/site.yaml`)
generates three surfaces:

```
data/site.yaml  ──►  docs/resource-library.md   (the Markdown guide)
                ──►  data/tools.json             (flat, filterable export)
                ──►  site/                        (the searchable web app)
```

- **Search** across 364 audited entries
- **Filter** by free / paid, EU / APAC, favourites, new-in-2026, acquired
- **Deep-link** to any section or filter state (shareable URLs)
- **Compare** up to 4 tools side by side
- **Tagged** automatically: pricing, region, status, favourite, format

Every link is checked **weekly** by an automated workflow that opens an issue when
something breaks — so the library no longer rots.

To add or change a resource, edit `data/site.yaml` (not the Markdown — it's generated).
See [docs/maintaining-the-library.md](docs/maintaining-the-library.md).

## 👋 If you're brand new to sourcing

Don't try to learn everything at once. Seriously. Start with:

1. Google X-ray searching
2. Boolean basics
3. LinkedIn Recruiter
4. Outreach fundamentals
5. One sourcing workflow you can repeat consistently

You do **not** need 75 Chrome extensions, an AI agent army, seven enrichment platforms,
or a second brain built in twelve apps. Most great sourcing is still curiosity,
persistence, pattern recognition, communication, and knowing where to look. Learn the
fundamentals first, then layer on tooling intentionally.

## 🧠 A few opinions about recruiting technology

Technology does not replace recruiter judgment. The best recruiters aren't the people
with the most tools — they're the people who ask better questions, understand talent
markets deeply, communicate clearly, recognize patterns, build trust quickly, and think
in systems instead of transactions.

AI will change recruiting. But the recruiters who win long-term will look less like
"prompt engineers" and more like talent strategists, workflow architects, market
interpreters, relationship builders, and information synthesizers.

Tools matter. But thinking matters more. → [More in the Field Guide](docs/field-guide.md)

## How to use this repo

### 1. Start with a use case

Don't try to consume the whole thing at once. Start with a specific question:

- I need better X-ray search engines.
- I want Chrome extensions for recruiter productivity.
- I need tools for technical sourcing.
- I want to audit my current sourcing stack.
- I need to map a talent market.
- I want AI to help with research, outreach, or intake.
- I'm building a sourcing training resource for my team.

### 2. Pick the right section

| If you need to... | Start here |
| --- | --- |
| Search and filter recruiting tools | [Searchable Library](https://ohsusannamarie.github.io/talent-sourcing-toolkit/) |
| Browse resources by category | [Resource Library](docs/resource-library.md) |
| Find people across the web | Google CSE archive + sourcing workflows |
| Source engineers or technical communities | [GitHub Sourcing](docs/github-sourcing.md) |
| Evaluate browser tools | Chrome extension archive + [Productivity](docs/recruiter-productivity.md) |
| Build market context | [Talent Intelligence](docs/talent-intelligence.md) |
| Use AI without being reckless | [AI Recruiting](docs/ai-recruiting.md) + [Compliance & Ethics](docs/compliance-and-ethics.md) |
| Create team enablement | [Templates](templates) + [Workflow Recipes](workflows) |

### 3. Test before adopting

Some tools may be outdated, deprecated, acquired, abandoned, blocked by your company's
security policy, or risky from a privacy/compliance standpoint. Treat this as a
discovery library, not a pre-approved vendor list.

### 4. Build your own shortlist

The real value is not using everything. It's finding the few tools, searches, and
workflows that make your sourcing work faster, cleaner, smarter, or more human.

## Chrome extension safety checklist

Before installing any Chrome extension for recruiting or research work, check:

- When it was last updated
- Who built it, and whether the developer looks legitimate
- Number and quality of reviews
- Requested permissions, and whether it can read or change data on websites
- Whether it touches LinkedIn, email, cookies, contacts, or candidate data
- Whether your company allows it
- Whether there is a privacy policy

A useful sourcing tool is not useful if it turns into a security problem.

---

## AI recruiting caution

AI can help recruiters research faster, draft cleaner, summarize intake notes, expand
search strings, compare markets, and personalize outreach. It should not be treated as
a neutral decision-maker. Be especially careful with tools that rank candidates, score
resumes, recommend who advances, reject automatically, infer protected traits, or
scrape/enrich candidate data without clear consent.

Use AI as decision support, not decision replacement.
